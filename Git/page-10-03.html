Git-Flow intuitif grâce à Tower

1. Client Git et Git-Flow

Tower est un client Git sous forme d’interface graphique. Il est conçu pour être très ergonomique et intègre des fonctionnalités très intéressantes. Au moment où ces lignes sont écrites, Tower est vendu 59 euros hors taxes et propose une réduction pour les étudiants.

Il n’existe pas d’équivalent intégrant Git-Flow de manière aussi complète sous Windows. Il existe cependant des clients Git très efficaces compatibles Windows comme TortoiseGit, SmartGit (qui intègre Git-Flow), SourceTree et GitHub Desktop.

Le contenu du projet n’a aucune importance ici. Le seul but est d’obtenir un historique composé de différentes branches représentatives de la méthode Git-Flow.

2. Cas pratique d’utilisation

Le dépôt résultant de ce cas pratique se trouve dans les fichiers téléchargeables : 08/git-flow.

Pour créer un nouveau dépôt avec Tower, il faut cliquer sur le bouton Create à partir de la page listant les dépôts utilisés :

Une fois que le dépôt a été créé, Tower propose de l’ouvrir avec le bouton Open de la fenêtre suivante :

La fenêtre ci-dessus présente des informations intéressantes sur le dépôt comme le chemin du dépôt, la branche sur laquelle est placée la référence HEAD ou encore le statut des fichiers. Une fois que le dépôt est ouvert dans Tower, il faut ajouter le premier commit qui va former le début de la branche master.

Pour ajouter le premier commit, il faut créer des fichiers dans un projet avec un autre logiciel que Tower (car Tower est un client Git et non un éditeur de texte). Une fois que les fichiers ont été ajoutés dans le projet, il faut cliquer sur le bouton Working Copy à gauche dans l’interface de Tower :

Cette interface permet d’effectuer un commit. Le fichier README n’est pas encore suivi par Git, son statut indique donc ?. Appuyer sur la checkbox à gauche du point d’exclamation permet d’ajouter le fichier à l’index.

Pour terminer le commit, il faut spécifier le message de commit, dans cet exemple C1 (master), puis cliquer sur le bouton Commit. Il est possible d’afficher ce commit en cliquant sur le bouton master à gauche.

Cette interface permet d’afficher les commits présents dans la branche master. Pour activer le mode Git-Flow dans Tower, il faut cliquer sur le bouton Git-Flow en haut de la fenêtre :

Il faut ensuite cliquer sur Enable Git-Flow. Après avoir cliqué sur ce bouton, Tower affiche une liste d’options de configuration propres à Git-Flow :

Cette fenêtre permet de configurer les branches éternelles (branche de production et branche de développement). Il est également possible de configurer les préfixes des branches temporaires ainsi que les préfixes des tags de version. Dans cet exemple, toutes les options seront laissées avec leur choix par défaut.

Pour créer le premier commit de la branche develop, il faut se placer dessus. Pour cela, il faut faire un clic droit sur le bouton develop à gauche puis cliquer sur Check Out "develop" :

Pour confirmer le changement de branche, Tower déplacera la référence HEAD à côté du bouton develop. Pour démarrer une nouvelle branche de fonctionnalité, il faut cliquer sur le bouton Git-Flow puis sur le bouton Start Feature.

Tower créera alors une branche de fonctionnalité et l’ajoutera à la liste des branches dans le menu de gauche de Tower.

Il faut ensuite effectuer des modifications dans le répertoire de travail et commiter pour que la branche existe réellement. Dans cet exemple, une branche de correctif et une branche de version sont également créées via le bouton Git-Flow et contiennent un commit. Une fois que ces branches ont été créées, l’affichage de l’historique (bouton History) ressemble à la capture suivante :

Pour terminer les branches éphémères, il faut cliquer sur le bouton Git-Flow puis cliquer sur Finish Feature. La fenêtre suivante s’affiche :

Plusieurs options s’offrent au développeur. L’une d’elles (Always Create Merge Commit) permet de respecter Git-Flow à la lettre en obligeant Git à créer un commit même dans le cas où une avance rapide aurait suffi. En fermant toutes les autres branches éphémères, Tower affiche ensuite l’écran correspondant à la capture suivante :

Les merges des branches sont visibles et permettent de bien suivre l’évolution de l’historique. Par exemple, la branche hotfix a correctement été mergée dans la branche master et dans la branche develop.

Cet exemple permet de mieux visualiser l’enchaînement des branches. Avec sa gestion du Git-Flow, Tower permet de simplifier son utilisation grâce à une interface intuitive.
