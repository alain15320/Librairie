Détail des chapitres

Le chapitre Git et la gestion de version présente les différents types de systèmes de gestion de versions qui ont existé. Il introduit également le fonctionnement et les intérêts des systèmes de gestion de versions.

Le chapitre Installation de Git explique l’installation de Git sur les trois systèmes d’exploitation principaux.

Le chapitre Création d’un dépôt détaille la manière de créer un dépôt (conteneur du code source d’un projet). Il explique également comment configurer le dépôt.

Le chapitre Manipulation des fichiers et commit présente les différentes façons de manipuler les fichiers à l’intérieur d’un dépôt. Le lecteur apprendra comment enregistrer ses modifications dans le dépôt.

Le chapitre Consultation et manipulation de l’historique explique au lecteur comment se servir du dépôt comme d’un véritable journal de bord du projet. Le lecteur apprendra à retrouver les modifications ayant eu lieu et à les manipuler. C’est au cours de ce chapitre que Git n’apparaîtra plus comme une contrainte pour le lecteur, mais comme une opportunité de mieux maîtriser son projet.

Le chapitre Les branches et les tags présente le fonctionnement des branches. Les branches représentent de nouveaux développements et permettent de séparer facilement différents développements en cours (production, développement, préproduction, etc.).

Le chapitre Partager un dépôt explique comment partager un dépôt avec d’autres collaborateurs pour facilement partager les développements entre développeurs. 

Le chapitre Git-Flow : workflow d’entreprise présente un système de gestion des branches : git-flow. Ce système, normalisé, permet de rationaliser la gestion des branches et de mutualiser un certain nombre de bonnes pratiques.

Le chapitre Les outils de Git explique brièvement le fonctionnement de plusieurs outils internes à Git. Le lecteur apprendra à gérer des bibliothèques dans son projet ou encore à retrouver le moment exact où une régression a été ajoutée au projet.

Le chapitre Scénario de développeur indépendant présente un scénario de développeur indépendant. Ce scénario présente la manière suivant laquelle Git s’utilise au quotidien et comment les commandes s’enchaînent les unes après les autres. Il présente également comment partager un dépôt sur Bitbucket et comment lier plusieurs dépôts entre eux via un dépôt central.

Le chapitre Scénario d’équipe présente un scénario d’équipe de développement. Il présente également un cas (presque) réel de l’utilisation de git-flow pour la gestion des branches. L’équipe de ce scénario utilise GitLab pour centraliser son dépôt et intégrer les développements des différents intervenants.

Un aide-mémoire est présent en annexe du livre. Il donne une liste des principales commandes et de leurs principales options pour utiliser Git efficacement au quotidien.
