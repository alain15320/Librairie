Accessibilité

La POO ajoute le principe d’accessibilité (visibilité) qui consiste à contrôler l’accès aux classes et à leurs membres (propriétés et méthodes).

Le terme d’accessibilité servant également à désigner les techniques d’utilisation d’une application pour des utilisateurs handicapés, le terme de visibilité sera préféré dans cette section.

Les niveaux de visibilité sous Java sont les suivants :

  • Classes :

  • public : les classes avec une visibilité publique sont accessibles par tout le monde (toutes les classes de tous les packages).

  • <rien> : le fait de ne pas mettre de visibilité à la classe rend celle-ci inaccessible en dehors des classes de son propre package.

  • private : ce niveau ne concerne que les classes internes. Ce sont des classes définies à l’intérieur d’une classe et qui ne sont utilisées que par celle-ci.

  • Membres :

  • public : les membres (propriétés et méthodes) sont accessibles à l’extérieur de la classe. Pour protéger les propriétés de modifications non autorisées ou malencontreuses, il est cependant fortement recommandé de ne jamais les déclarer publiques.

  • protected : les membres sont accessibles aux classes descendantes (voir la notion d’héritage plus loin) mais aussi avec Java aux autres classes du même package.

  • <rien> : les attributs et méthodes sont accessibles uniquement pour les classes appartenant au même package si aucune visibilité n’est fournie.

  • private : c’est le niveau le plus élevé de restriction. Les propriétés et méthodes ne sont alors accessibles que depuis la classe elle-même. Concernant les propriétés, elles peuvent l’être en dehors de la classe mais uniquement par le biais de ses mutateurs et accesseurs accessibles.

Les membres (attributs et méthodes) d’une classe peuvent également être qualifiés avec des mots-clés particuliers :

  • static : les membres déclarés static appartiennent à la classe et non aux objets issus de la classe. Ils ne peuvent pas être hérités par les classes filles.

  • final : les attributs déclarés final ne peuvent plus être modifiés une fois initialisés. Ils doivent également être initialisés à la construction des objets. Les méthodes déclarées final ne peuvent plus être redéfinies dans les classes filles.

D’autres mots-clés permettent de caractériser les classes et leurs membres dans des domaines autres que celui de la visibilité tels que la concurrence ou l’optimisation. Les lecteurs curieux chercheront les usages de volatile, synchronized et transient.

Voici la classe réécrite en tenant compte des principes énoncés.

public class Chat {         // propriétés         private String nom;         private String couleur;         private long dateDeNaissance;                 // accesseurs <=> getters         public String getNomChat() {           return nom;         }         public String getCouleur() {              return couleur;         }         public int getAge() {              // calcule l’âge à partir de la date de naissance             return calculerAge(dateDeNaissance) ;        }                  public int getDateDeNaissance() {              return dateDeNaissance ;        }                  // mutateurs <=> setters         public void setNom(String nom) {           this.nom = nom;         }              public void setCouleur(String couleur) {           this.couleur = couleur;         }             protected void setDateDeNaissance(long nouvelleDate) {           this.dateDeNaissance = nouvelleDate;         }                  // autres méthodes         public void dormir(){             }         public void manger(Object deLaNourriture){             }         public String miauler(){           String leMiaulement = "miaou";           return leMiaulement;         }            private static int calculerAge(long date) {             Instant naissance = Instant.ofEpochMilli(date) ;             Instant maintenant = Instant.now() ;             long annees = ChronoUnit.YEAR.between(naissance, maintenant) ;            return annees ;        }   }

L’attribut dérivé age discuté précédemment est alors défini par l’utilisation de la méthode getAge(). Il ne correspond pas à une propriété stockée dans l’objet, mais à un calcul qui fait intervenir un autre attribut.

Toute application doit disposer d’une classe avec une méthode particulière chargée de son exécution. Il s’agit de la méthode main() qui a été rencontrée au chapitre Prise en main d’Eclipse. Sa syntaxe et sa structure sont toujours les mêmes :

public static void main(String[] args) {       // action(s)    }

Par convention, le terme args est employé mais il est possible d’en choisir un autre.

Il s’agit d’une méthode publique et statique. Comme vu précédemment, elle est donc accessible à l’extérieur de la classe et appartient à celle-ci et non à ses instances.

La méthode main() prend un paramètre obligatoire nommé args de type String[].

Exemple :

public static void main(String[] args) {       System.out.print("Java, c’est extra !");    }
