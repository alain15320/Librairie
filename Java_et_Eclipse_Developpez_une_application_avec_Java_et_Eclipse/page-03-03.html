Découverte de l’IDE

Accédons sans plus tarder au workbench ou plan de travail.

 Cliquez sur l’icône suivante .

1. Perspective - Vue

Eclipse propose un environnement de développement par défaut nommé perspective, composé de plusieurs onglets (des vues dans la terminologie Eclipse). Plus précisément, il s’agit de la perspective Java.

Il existe d’autres perspectives accessibles par le menu Window - Open Perspective et qui sont utilisées selon les besoins des développeurs. Pour ne pas encombrer l’espace mémoire de l’IDE, il vaut mieux fermer celles qui ne sont pas utilisées.

Chaque perspective est constituée d’un certain nombre d’éléments appelés vues (views) qui ne sont pas forcément tous ouverts (visibles). Comme pour les perspectives, n’ouvrez que les vues qui vous sont réellement utiles, sinon vous risquez de travailler sur une partie de l’écran aussi grande qu’un timbre-poste.

 Pour ajouter par exemple la vue Console à la perspective Java, choisissez l’option du menu Window - Show View - Console.

 Pour afficher une vue dans tout l’espace disponible, double cliquez sur son onglet. Double cliquez une deuxième fois sur l’onglet pour restaurer la vue avec sa taille initiale.

 Pour restaurer une perspective, après un clic droit sur son nom dans l’onglet situé en haut à gauche de l’écran, choisissez Reset.

2. Éditeur de code

Il est possible de choisir entre deux éditeurs de code, Java Editor et Text Editor. 

Par défaut, les classes sont ouvertes avec l’éditeur Java.

Une instruction peut être longue et dépasser la largeur d’une page-écran. Pour l’écrire sur plusieurs lignes, il suffit d’effectuer un saut de ligne à l’endroit où vous voulez effectuer une coupure.

Exemple :

laFenetre.getJLab_Action().setIcon(new ImageIcon(
		Fenetre.class.getResource("/images/gestion/modifier.png")));

Par contre, pour couper les chaînes, il faut obligatoirement utiliser l’opérateur +. Ceci étant, Eclipse se charge de le faire pour vous dès que vous effectuez un saut de ligne.

 À titre d’exercice, placez le curseur au milieu d’une chaîne de caractères, et appuyez sur [Entrée].

La chaîne sera automatiquement coupée en deux, sur deux lignes différentes.

Exemple :

  ...
getResource("/images/gestion" +
					"/modifier.png")));

 Pour bénéficier de l’autocomplétion, saisissez les premières lettres suivies de la combinaison de touches [Ctrl][Espace].

Eclipse complète votre saisie ou vous propose une liste. Vous pouvez paramétrer l’autocomplétion en modifiant les options disponibles ici : Window - Preferences - Editor - Content Assist.

 Pour provoquer volontairement une indentation, sélectionnez la ou les lignes contiguës concernées et utilisez le raccourci-clavier [Ctrl] I ou faites un clic droit et choisissez Source - Correct Indentation.

Vous noterez que dans le menu contextuel les commandes disponibles sont accompagnées sur leur droite des raccourcis-clavier associés.

Eclipse ne formate pas automatiquement le code par défaut.

 Pour formater le code (espaces, indentation…), utilisez le raccourci-clavier [Ctrl][Shift] F, ou faites un clic droit et sélectionnez Source - Format.

Pour ne formater qu’une partie du code, sélectionnez celui-ci et effectuez l’opération précédente.

 Pour désactiver le correcteur orthographique qui agit sur vos commentaires, choisissez Window - Preferences - General - Editors - Text Editors - Spelling puis décochez l’option Enable spell checking.

 Pour ajouter les numéros de lignes, développer ou réduire le code, effectuez un clic droit sur la bande verticale gauche légèrement grisée.

3. Débogueur

Le développement nécessite de nombreux tests. Pour contrôler la bonne exécution d’une partie d’un programme comportant des structures conditionnelles et itératives, l’utilisation d’un débogueur s’avère souvent indispensable. La découverte du débogueur d’Eclipse se fera au travers d’un programme qui affiche les tables de multiplication.

Voici une proposition de code pour un tel programme :

 Testez auparavant le programme en appuyant sur les touches [Ctrl][F11].

 Effectuez un clic droit dans la marge grise devant le numéro de ligne 20 et choisissez Toggle Breakpoint.

 Testez le programme en appuyant cette fois uniquement sur la touche [F11] pour lancer le débogueur.

Si la perspective Debug n’est pas encore ouverte, Eclipse affiche une fenêtre.

 Cochez l’option Remember my decision.

Ceci a pour conséquence d’ouvrir cette perspective ; un bouton est alors ajouté à la barre principale des icônes de l’IDE pour permettre d’y accéder rapidement.

Les valeurs des variables sont affichées dans une vue.

Le débogueur va permettre de vérifier l’exactitude des données en contrôlant pas à pas les valeurs des variables.

 Cliquez sur l’icône ou appuyez sur la touche [F5].

Le point d’arrêt étant à la ligne 20, le parcours concerne la boucle for interne. À l’itération suivante, la variable ligne doit contenir le nombre 1 et les variables colonne et nombre doivent demeurer inchangées.

 Pour poursuivre les tests et lire les valeurs des variables, appuyez à nouveau sur [F5].

 Pour revenir à la perspective Java, cliquez sur le bouton Java en haut à droite de la fenêtre principale de l’IDE.

Si celui-ci n’est pas visible, cliquez sur les doubles chevrons pour le faire apparaître.
