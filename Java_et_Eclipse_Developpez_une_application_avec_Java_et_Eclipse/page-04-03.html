Première classe

 Sélectionnez le package qui a été créé.

 Passez par le menu File - New - Class ou effectuez un clic droit puis New - Class.

La boîte de dialogue de création de classe s’ouvre.

 Nommez la classe MonPremierProgramme en commençant le nom par une majuscule (pour respecter les conventions de nommage de Java) et cochez l’option public static void main(String[] args). Cette option permet de créer la méthode servant de point d’entrée à l’exécution du programme.

Dans les conventions de Java, seules les classes commencent par une majuscule. Les packages, attributs, méthodes et variables commencent par une minuscule.

Les caractères spéciaux (%, ?…) et les chiffres ne peuvent pas être utilisés en tant que premier caractère.

 Cliquez sur le bouton Finish.

La classe a été ajoutée au package et le code généré automatiquement par Eclipse est maintenant visible dans l’éditeur de code.

 Ajoutez une propriété (appelée aussi attribut) privée à la classe.

package priseEnMain.premier;

public class MonPremierProgramme {

	private String prenom;

	public static void main(String[] args){
			// TODO Auto-generated method stub
	}
}

Une des petites règles de programmation de l’auteur de ces lignes est de toujours commencer par déclarer les propriétés ou les méthodes comme privées. S’il est nécessaire de pouvoir y accéder depuis l’extérieur de la classe, la visibilité est modifiée. Cela permet d’améliorer l’encapsulation des données et peut se résumer par l’adage « d’abord interdire, puis autoriser  ».

Il s’agit ensuite d’ajouter des accesseurs et mutateurs de cette propriété dans la classe.

 Choisissez dans le menu clic droit Source - Generate Getters and Setters…. Cochez ensuite la case prenom (les autres cases sous la variable prenom se cochent alors automatiquement) puis validez.

Les accesseurs et mutateurs sont générés.

public String getPrenom() {
	return prenom;
}

public void setPrenom(String prenom){
	this.prenom = prenom;
}

 Ajoutez un constructeur dans cette classe en faisant à nouveau un clic droit dans l’éditeur de code et en sélectionnant l’option Source - Generate Constructor using Fields….

 Cliquez sur OK pour valider. Le constructeur est généré.

public MonPremierProgramme(String prenom) {
	super();
	this.prenom = prenom;
}

L’instruction super() appellera le constructeur de la super-classe, ici java.lang.Object. 

Eclipse initialise automatiquement la propriété prenom parce que la case équivalente était cochée dans la boîte de dialogue précédente. Les noms de la propriété et de la variable sont les mêmes : pour les différencier, la propriété de la classe est préfixée par this suivi d’un point.

Pour ajouter des commentaires dans la classe, ajoutez deux slashes (/) au début de la ligne ou là où vous voulez commenter le reste de la ligne. Tout ce qui est en commentaire ne sera pas traité par le compilateur.

Pour créer des commentaires sur plusieurs lignes, encadrez-les par /* et */.

Si les mots TODO ou FIXME sont écrits dans un commentaire, la ligne sera indexée dans Eclipse. C’est un moyen pratique pour laisser des indications sur ce qui reste à faire ou à corriger, à la fin d’une dure journée de labeur.

La ligne TODO Auto-generated method stub est ajoutée par Eclipse quand une méthode est générée automatiquement. Il est préférable de l’enlever ou d’y substituer vos propres commentaires.

Il est possible d’ajouter des commentaires particuliers commençant par /** et finissant par **/. Ces commentaires dits javadoc seront utilisés pour générer la documentation de l’application.

 Tapez /** au-dessus de la méthode main et appuyez sur [Entrée].

Le patron de commentaire javadoc est automatiquement créé par Eclipse.

 Complétez la méthode main().

public static void main(String[] args) {
	MonPremierProgramme programme = new MonPremierProgramme("Moi");
	System.out.println("C’est " +programme.prenom +"!");
}

Il s’agit ensuite de tester ce qui vient d’être programmé.

Pour tester ce programme, il est possible de procéder de plusieurs manières :

  • Effectuez un clic droit puis choisissez Run As - Java Application.

  • La classe étant ouverte dans l’éditeur, appuyez sur les touches [Ctrl][F11].

  • Cliquez sur l’icône dans la barre d’outils d’Eclipse.

Le programme s’exécute alors et la sortie est affichée dans la vue console.

De nombreuses notions propres à la programmation orientée objet (POO) sont déjà présentes dans ces quelques lignes : package, propriété, constructeur, getter, setter…

Ces notions seront revues au chapitre Concepts de base de la POO et lors de la réalisation du projet.

À ce stade, retenez les points suivants :

  • Un programme comporte au moins une classe.

  • La classe doit porter le même nom que le fichier dans laquelle elle se trouve.

  • Les minuscules et majuscules sont considérées comme des caractères différents : Java est sensible à la casse.

  • Toute classe doit se trouver dans un package. Un package peut contenir d’autres packages.

  • Le plus souvent, la classe possède un constructeur explicite. Ce constructeur permet de créer des instances (on parle d’instanciation) de la classe, ces instances étant également appelées objets.

  • Toutes les méthodes proviennent obligatoirement d’une classe : soit une classe créée par le développeur, soit une classe fournie par l’API Java, soit une classe fournie par une librairie créée par d’autres développeurs.

  • La méthode principale main() permet d’exécuter la classe. Il s’agit du point d’entrée de l’application : elle est toujours appelée en premier.

  • Une classe commence par une majuscule, une variable commence par une minuscule.

Rien n’interdit de créer plusieurs classes disposant de la méthode main(). Il devient alors nécessaire de désigner celle dont la méthode main() sera exécutée, les autres étant alors ignorées.

Essayez de ne créer qu’une seule méthode main() par projet. Cela permet d’éviter les ambiguïtés.
