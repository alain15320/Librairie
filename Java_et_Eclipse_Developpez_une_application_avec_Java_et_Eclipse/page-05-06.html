Méthode

Les méthodes représentent les actions et les comportements que peuvent mener des objets issus d’une classe. Elles définissent l’aspect dynamique ou comportemental de la classe. Les chats peuvent dormir, manger, miauler, etc.

Voici quelques-unes de ces méthodes.

class Chat {           // propriétés          String nom;          String couleur;            long dateDeNaissance;              // méthodes           void dormir(){                return ;         }            void manger(Object deLaNourriture){          }              String miauler(){                String leMiaulement = "miaou";               return leMiaulement;          }    }

Les méthodes qui ne renvoient aucune valeur correspondent à des procédures. Elles sont toujours précédées du mot-clé void.

Les méthodes qui renvoient une valeur correspondent à des fonctions. Elles sont toujours précédées du type de la valeur renvoyée.

Les méthodes peuvent bien sûr contenir des paramètres qui doivent tous être typés.

Le mot-clé return permet de sortir de la méthode et de retourner à la méthode appelante.

Exemple :

   double pourcentage(Double numerateur, Double denominateur) {         Double resultat = numerateur/denominateur * 100;           return resultat;             }

Le code ci-dessus comporte des actions d’autoboxing implicites, notion qui sera détaillée dans le prochain chapitre.

Le nom d’une méthode et les types ordonnés de ses paramètres constituent sa signature (le type de retour de la méthode est exclu de la signature, tout comme les noms des paramètres). À l’intérieur d’une classe donnée, il ne peut exister qu’une seule méthode par signature.

Par exemple, le code suivant provoquera des erreurs à la compilation.

public class MauvaiseSignature {        void calculer(int a, int b) {               }       // le compilateur n’accepte pas cette méthode       int calculer(int c, int d) {             return c + d ;       }   }

Parmi les méthodes, certaines ont pour rôle de retourner les valeurs des propriétés de la classe et d’autres de les modifier. Elles sont nommées respectivement accesseurs et mutateurs (getters et setters).

Exemple :

int getAge() {       // retourne l’âge calculé à partir de la        // date de naissance    }       long getDateDeNaissance() {       return dateDeNaissance ;   }       void setDateDeNaissance(long nouvelleDate) {       dateDeNaissance = nouvelleDate ;   }

Ces accesseurs et mutateurs sont traditionnellement codés sous la forme getXXX/setXXX. Le mélange français-anglais n’est pas idéal ici, mais beaucoup d’outils reposent sur cette notation pour découvrir dynamiquement les propriétés d’une classe donnée, comme les Java Beans. Il est fortement conseillé de garder cette habitude de notation.

Le terme de services est parfois utilisé pour désigner les méthodes et celui d’opération pour l’implémentation des méthodes.

Des méthodes peuvent être préfixées avec le mot-clé static. Elles ne sont alors plus spécifiques à l’objet lui-même mais à sa classe. Cela permet le plus souvent de créer des services utilitaires, et plus communément le point d’entrée d’une application.

Exemple :

static int calculerAge(long date) {        Instant naissance = Instant.ofEpochMilli(date) ;       Instant maintenant = Instant.now() ;       long annees = ChronoUnit.YEAR.between(naissance, maintenant) ;      return annees ;   }

Les méthodes statiques sont très utiles mais ont tendance à « casser » le paradigme objet, et à rendre le code plus difficile à modifier. Il est conseillé de les créer avec parcimonie.
