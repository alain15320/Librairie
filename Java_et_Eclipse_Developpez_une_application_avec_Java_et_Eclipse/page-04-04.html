Première fenêtre

Une petite application avec une interface graphique (GUI pour Graphical User Interface en anglais, IHM pour interface homme-machine en français) va maintenant être créée.

Une des nouveautés d’Eclipse depuis la version Indigo est de fournir un éditeur d’interface graphique issu du monde Google, WindowBuilder. Il en existe d’autres mais compte tenu de la qualité de WindowBuilder ce dernier sera utilisé pour le projet. Quant au plug-in Visual Editor, il est abandonné.

WindowBuilder est configurable via le menu Window - Preferences. Les options par défaut sont utilisées pour le projet.

WindowBuilder est un plug-in d’Eclipse qui permet de créer rapidement des interfaces graphiques en WYSIWYG (What You See Is What You Get : Ce que vous voyez est ce que vous obtenez).

Si vous voulez que le code généré automatiquement ait une forme différente, vous pouvez paramétrer WindowBuilder en naviguant à partir du menu Window - Préférences - WindowBuilder.

Tous les créateurs d’interfaces graphiques ont la même caractéristique : ils permettent de créer rapidement des composants graphiques. Par contre, ils ne fourniront quasiment jamais directement ce que vous voulez : il sera donc nécessaire d’intervenir ensuite sur le code généré.

1. Création de la fenêtre

L’objectif est de créer une fenêtre très simple en s’aidant de WindowBuilder.

Cette fenêtre comportera un champ de saisie textuelle dans lequel l’utilisateur pourra saisir des mots. Un clic sur le bouton OK de la fenêtre présentera ce qui aura été saisi dans une autre fenêtre.

 Faites un clic droit sur le projet dans l’explorateur de packages et choisissez New - Other.

 Ouvrez les dossiers WindowBuilder et Swing Designer puis sélectionnez JFrame.

Une JFrame est une fenêtre graphique autonome, susceptible de contenir d’autres éléments graphiques. Il s’agit du conteneur graphique de cette application. 

Il est possible de créer d’autres conteneurs graphiques, notamment JDialog qui est utilisé pour créer des boîtes de dialogues et JPanel qui est un conteneur d’éléments graphiques qui n’a pas de fenêtrage.

 Nommez la classe PremiereFenetre et choisissez le package priseEnMain.graphique comme package de destination.

Comme pour la classe PremierProgramme, le code est généré automatiquement et apparaît dans la fenêtre de l’éditeur.

En dessous du code de l’éditeur, deux onglets apparaissent : Source et Design. Source sert à visualiser et modifier le code source en mode textuel, Design sert à visualiser et modifier la fenêtre en mode graphique.

Si les onglets n’apparaissent pas, il est possible de forcer l’ouverture de la classe avec l’éditeur de WindowBuilder de la manière suivante :

 Sélectionnez la classe PremiereFenetre dans l’explorateur de packages, faites un clic droit et sélectionnez Open With - WindowBuilder Editor.

 Cliquez ensuite sur l’onglet Design.

Voici une vue d’ensemble de l’environnement de développement :

La fenêtre créée contient un composant particulier de type JPanel, qui est un conteneur dans lequel il est possible de placer d’autres composants graphiques, tels que des boutons, des champs de saisie, des tables, des listes ou d’autres JPanel. 

Ce composant JPanel est la base graphique de l’application, la fenêtre se chargeant avant tout de l’inclure dans le système de fenêtrage du système d’exploitation et permettant à l’application d’avoir une icône, un titre, des boutons de maximisation, minimisation, fermeture, et lui procurant une position et une taille sur l’écran de l’ordinateur.

 Cliquez dans le centre de la petite fenêtre de l’éditeur. Cela sélectionne le panneau principal.

 Pour accéder au code du composant sélectionné, cliquez alors sur l’onglet Source.

 Lancez la fenêtre avec [Ctrl][F11].

2. Ajout de composants

Une fenêtre vide n’est pas très utile, ni très attrayante. Des composants graphiques (appelés également widgets) vont lui être ajoutés, ainsi que quelques petits éléments d’habillage, dont :

  • un titre,

  • une icône,

  • une zone de saisie,

  • un bouton de validation.

 Choisissez dans votre disque dur une image, si possible faisant 32 pixels par 32 pixels.

 Créez un dossier contenant vos ressources graphiques, en faisant un clic droit puis en sélectionnant New - Source Folder. Saisissez src/main/resources comme nom de dossier.

Le caractère / sera automatiquement reconnu par Eclipse (et par Java) comme un séparateur de nom de dossier, et cela sur toutes les plateformes !

 Créez un package priseEnMain.graphique dans ce nouveau dossier. Il doit avoir le même nom que le package de la classe graphique.

 Copiez votre image dans le package priseEnMain.graphique de ce dossier src/main/resources.

 Dans l’arbre des widgets à la section Components, cliquez sur (javax.swing.JFrame).

La section Properties située en dessous se met à jour pour afficher les propriétés de la fenêtre.

 Cliquez sur le petit bouton avec trois points en face de la propriété iconImage. 

Une boîte de dialogue permettant de choisir l’image apparaît.

 Cliquez sur le bouton radio Classpath resources, et ouvrez le dossier resources dans la section Parameters jusqu’à faire apparaître votre image. Double cliquez dessus.

La fenêtre a dorénavant une icône dans sa barre de titre.

 Dans la section Properties, faites défiler verticalement les propriétés jusqu’à voir la propriété title. Cliquez en face du nom de la propriété pour changer sa valeur.

La fenêtre a maintenant un titre.

L’étape suivante consiste à s’intéresser à l’intérieur de la fenêtre.

 Cliquez sur le composant JPanel soit dans la section Components, soit dans la prévisualisation à droite.

La section Properties affiche les propriétés du panneau.

Le panneau est un conteneur graphique, et tout conteneur doit savoir comment placer ce qui est contenu à l’intérieur.

Java permet de spécifier les règles de ce placement à l’aide d’un layout, ou disposition. Le layout par défaut d’un Panel est le BorderLayout.

 Changez ce layout par un Absolute layout à l’aide de la propriété du panneau Properties.

Ce layout permet de positionner les éléments du panneau au pixel près.

Ce layout est utile mais devrait être réservé à quelques cas très particuliers, car il ne gère pas bien le redimensionnement des fenêtres.

 Dans la palette, cliquez sur le composant JButton. Déplacez ensuite la souris sur la prévisualisation et déposez-le plutôt vers le bas de la fenêtre. Redimensionnez-le et changez sa propriété text dans la section Properties.

 De la même manière, sélectionnez, déposez et redimensionnez un composant JTextField pour ajouter un champ de saisie à la fenêtre.

 Sélectionnez, déposez et redimensionnez un composant JLabel. Ce composant permet d’afficher des textes non éditables et des images.

 Changez ses propriétés : police, taille de caractères, alignement (par exemple centré), couleur...
