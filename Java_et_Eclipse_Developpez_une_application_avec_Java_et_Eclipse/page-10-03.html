Installation et configuration du serveur XAMPP

Pour une installation ou une mise à jour, vous pouvez vous rendre sur différents sites de téléchargement. Plusieurs kits d’installation sont disponibles. Le plus simple est celui comportant un installateur. La version utilisée lors de l’écriture de cet ouvrage est XAMPP 5.6.8-0-VC11 pour win32.

Les explications et les copies d’écrans qui suivent concernent cette version. Les opérations à mener pour des versions plus récentes peuvent être différentes.

 Double cliquez sur l’icône. L’installation se fait automatiquement.

 Sur Windows 8, un avertissement apparaît. Cliquez sur OK.

 Vous pouvez ensuite choisir les composants à installer. Laissez les options par défaut. Si vous n’avez pas assez de place sur votre disque dur, ne cochez que Apache, MySQL et phpMyAdmin.

 Le dossier par défaut pour l’installation est C:\xampp. Vous pouvez choisir un autre dossier. Cliquez ensuite sur le bouton Next.

 Continuez jusqu’à aboutir à cet écran. Cliquez alors sur Next pour lancer le processus d’installation.

 Si tout s’est bien passé, vous devriez aboutir à l’écran suivant. Cliquez sur le bouton Finish.

 Accédez au panneau de contrôle en bas à droite de votre écran puis cliquez sur les boutons Start pour lancer Apache et MySQL.

 Cliquez ensuite sur le bouton Admin de l’option MySQL.

Une page web est ouverte, commençant par l’adresse http://localhost/phpmyadmin. Il s’agit de la page d’administration des bases de données MySQL gérées par phpMyAdmin.

1. Gestion de l’accès au serveur

Lors de la première utilisation, l’accès à XAMPP ne comporte pas de mot de passe. Celui-ci n’affiche pas de message pour prévenir des risques. Un accès non sécurisé n’est pas acceptable pour une application et ses données.

Le premier travail à effectuer est donc de sécuriser l’accès au serveur.

 Cliquez sur l’icône d’accueil puis sur le bouton Utilisateurs.

 Cliquez sur le lien Changer les privilèges.

 Cliquez ensuite sur le bouton Modifier le mot de passe.

 Saisissez un mot de passe puis cliquez sur le bouton Exécuter. Celui qui est utilisé pour le projet Luna est « tempo » (ne pas saisir les guillemets).

Après la création du mot de passe, un message apparaît.

 Arrêtez tous les services en cliquant sur les boutons Stop dans le panneau de contrôle puis quittez celui-ci en cliquant sur le bouton Exit.

Les paramètres de l’utilisateur root ont été modifiés. Il faut ensuite mettre à jour ces données dans le config.inc.php pour qu’ils soient pris en compte lors de la prochaine ouverture du serveur.

 Ouvrez le fichier config.inc.php. Il se situe habituellement dans le dossier suivant : C:\xampp\phpMyAdmin.

 Recherchez la ligne $cfg[’Servers’][$i][’password’] et effectuez les modifications :

config.inc.php avant : $cfg[’Servers’][$i][’password’] = ’’;

config.inc.php après : $cfg[’Servers’][$i][’password’] = ’tempo’;

Ou mettez votre propre mot de passe à la place de "tempo".

 Relancez le serveur comme vu précédemment.

2. Création de la base de données MySQL

La base de données de l’application Luna est composée de plusieurs tables, dont les tables client, article, commande, lignescommandes, modereglements et categorie.

Auaparavant, il était nécessaire de créer ces tables manuellement dans la base de données. Avec JPA, le processus de création peut être entièrement automatisé. Il reste néanmoins nécessaire de créer la base elle-même.

 Lancez phpMyAdmin.

 Dans le menu Accueil, cliquez sur Nouvelle base de données.

 Dans le champ Nom de base de données, saisissez « luna », puis cliquez sur Créer.

Votre base de données est alors créée.

L’accès au serveur MySQL a été précédemment sécurisé en créant un mot de passe pour le super-utilisateur root. Au lieu d’accéder aux bases avec ce super-utilisateur, ce qui est fortement déconseillé, un utilisateur va être créé qui servira de point d’accès à la base « luna ».

 Cliquez sur Utilisateurs comme précédemment, puis cliquez sur Ajouter un utilisateur.

 Saisissez ensuite le nom d’utilisateur, par exemple « eni » (sans guillemets). Puis saisissez deux fois son mot de passe. « java » (toujours sans guillemets) est retenu. Ces valeurs sont arbitraires, il est juste important de bien les retenir car elles serviront à l’application plus tard.

 Assurez-vous que tous les privilèges globaux sont décochés. Cliquez enfin sur Exécuter. L’utilisateur est créé.

Il s’agit maintenant de lui donner des droits spécifiques d’accès sur la base « luna » et uniquement celle-ci.

 Cliquez sur le lien Base de données, et sélectionnez la base « luna ».

 Cliquez ensuite sur Exécuter. La page de privilèges spécifique à cette base s’affiche alors. Dans un souci de rapidité, cliquez sur Tout cocher, puis sur Exécuter.

Il reste alors une dernière étape pour permettre à l’application d’accéder à sa base de données. En effet, MySQL gère les privilèges d’accès à l’aide de différentes règles. La première règle correspondant à la demande d’accès est choisie pour garantir cet accès. Or par défaut lors de l’installation, deux règles interdisent cet accès : il s’agit des règles « anonymous » qu’il faut effacer.

 Dans l’onglet Utilisateurs, sélectionnez les deux premières règles. Puis cliquez sur Exécuter dans la section Effacer les utilisateurs sélectionnés.

L’utilisateur « eni » peut dorénavant accéder à la base de données « luna », ce qui va pouvoir être vérifié un peu plus loin avec le pilote ODBC.

Les opérations de configuration d’une base de données en production sont généralement déléguées à un administrateur de bases de données.
